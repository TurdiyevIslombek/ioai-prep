Section 01

What Is a Vector?

Forget everything complicated. A vector is just a list of numbers — an arrow that points somewhere. If you write v = [3, 2], that means "go 3 steps right and 2 steps up" from the origin.

But here's the key idea that 3Blue1Brown emphasizes: there are three ways to think about vectors, and you need all three depending on context:

🎓 Physics view

An arrow with a direction and length. Can be moved around — only the direction and magnitude matter.

💻 CS view

An ordered list of numbers. [3, 2] in 2D, [1, 0, -4, 7] in 4D. Each entry is a "feature."

📐 Math view

Anything that can be added together and scaled by a number, following certain rules (a "vector space").

🤖 ML view

A data point. Each image, sentence, or user is represented as a vector of numbers in some high-dimensional space.

Key insight For IOAI, the CS/math hybrid view dominates. A vector is a column of numbers that lives in ℝⁿ (the set of all lists of n real numbers). A 28×28 pixel grayscale image? That's a vector in ℝ⁷⁸⁴ — just flatten all 784 pixels into one long list.

Notation you must know

Vectors are written as column vectors by default in math:

Column vector in ℝ³ v = ⎡ 3 ⎤
    ⎢ 2 ⎥
    ⎣-1 ⎦

We write this compactly as v = [3, 2, -1]ᵀ where the ᵀ means transpose — flip the row into a column. The number of entries is the dimension. This vector is in ℝ³ (three-dimensional real space).

In Python (NumPy)

import numpy as np

# Creating vectors
v = np.array([3, 2, -1])          # Shape: (3,) — 1D array
v_col = np.array([[3], [2], [-1]])  # Shape: (3,1) — actual column vector

# Easier way to make a column vector
v_col = v.reshape(-1, 1)             # Same thing: reshape to (3,1)

print(v.shape)       # (3,)
print(v_col.shape)   # (3, 1)
Common beginner trap In NumPy, np.array([3, 2, -1]) has shape (3,) — it's neither a row nor a column. NumPy usually handles this fine, but in IOAI coding problems, shape mismatches cause silent bugs. Always check .shape when debugging.
Section 02

Vector Operations

Addition: tip-to-tail

To add two vectors, place the second one's tail at the first one's tip. The result goes from the origin to the second arrow's tip. Numerically: just add entry by entry.

Vector addition [3, 2] + [1, -1] = [3+1, 2+(-1)] = [4, 1]

Scalar multiplication: stretching

Multiply every entry by the same number (the "scalar"). This stretches or flips the vector.

Scalar multiplication 2 · [3, 2] = [6, 4]     (twice as long, same direction)
-1 · [3, 2] = [-3, -2]    (same length, opposite direction)
✏️ Work this out on paper right now:

Given a = [2, 5] and b = [-1, 3]:
1.Compute a + b = [2+(-1), 5+3] = [1, 8]
2.Compute 3a = [6, 15]
3.Compute 2a - b = [2·2-(-1), 2·5-3] = [4+1, 10-3] = [5, 7]
4.Draw all of these on graph paper. See how 2a - b is a linear combination.
# Vector operations in NumPy
a = np.array([2, 5])
b = np.array([-1, 3])

print(a + b)      # [1, 8]
print(3 * a)       # [ 6, 15]
print(2*a - b)     # [5, 7]

# Length (magnitude / norm) of a vector
print(np.linalg.norm(a))  # sqrt(4 + 25) = sqrt(29) ≈ 5.385

Dot product: how similar are two vectors?

The dot product (inner product) of two vectors multiplies them entry-by-entry and sums up:

Dot product a · b = a₁b₁ + a₂b₂ + ... + aₙbₙ

[2, 5] · [-1, 3] = (2)(−1) + (5)(3) = −2 + 15 = 13

The dot product has a geometric meaning:

a · b = ‖a‖ · ‖b‖ · cos(θ)

Where θ is the angle between the vectors. This tells you:

  • Positive dot product → vectors point in roughly the same direction (angle < 90°)
  • Zero dot product → vectors are perpendicular (orthogonal) — this is huge in ML
  • Negative dot product → vectors point in roughly opposite directions (angle > 90°)
Why IOAI cares The dot product is the foundation of ALL of ML. Neural networks compute dot products between inputs and weights. Cosine similarity (used in embeddings, search, recommendations) is just a normalized dot product. When you hear "attention mechanism in transformers" — it's dot products between query and key vectors. 
# Dot product in NumPy
a = np.array([2, 5])
b = np.array([-1, 3])

# Three equivalent ways:
print(np.dot(a, b))   # 13
print(a @ b)           # 13  (@ is the preferred way)
print(np.sum(a * b))   # 13  (element-wise multiply, then sum)

# Checking orthogonality
u = np.array([1, 0])
v = np.array([0, 1])
print(u @ v)  # 0 → perpendicular!

# Cosine similarity (used in embeddings)
def cosine_sim(a, b):
    return (a @ b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(cosine_sim(a, b))  # 0.762...
Section 03

Linear Combinations, Span & Independence

Linear combinations

A linear combination is what you get when you scale some vectors and add them together:

Linear combination c₁v₁ + c₂v₂ + ... + cₖvₖ

Where c₁, c₂, ... are scalars (any real numbers) and v₁, v₂, ... are vectors.

Example: Let v₁ = [1, 0] and v₂ = [0, 1] (the standard basis vectors).

Then 3v₁ + 2v₂ = 3[1,0] + 2[0,1] = [3, 0] + [0, 2] = [3, 2]

In fact, any vector [a, b] = a·[1,0] + b·[0,1]. So every 2D vector is a linear combination of these two basis vectors!

Span: what can you reach?

The span of a set of vectors is the set of ALL possible linear combinations of those vectors. Think: "what's the collection of every vector you can build by scaling and adding these vectors?"

Span of 1 vector

A line through the origin. You can only stretch/shrink along that one direction.

Span of 2 (independent) vectors

A plane. Like how î and ĵ span all of 2D space.

Span of 2 parallel vectors

Still just a line! The second vector adds nothing new — it's "redundant."

Span of 3 vectors in ℝ³

All of 3D space (if independent), or a plane (if one is redundant), or a line (if two are).

Linear independence: no redundancy

Vectors are linearly independent if none of them can be written as a linear combination of the others. Equivalently: the only way to get the zero vector from their linear combination is if ALL the scalars are zero.

Definition of linear independence c₁v₁ + c₂v₂ + ... + cₖvₖ = 0  ⟹  c₁ = c₂ = ... = cₖ = 0
Test: are these linearly independent?

v₁ = [1, 2], v₂ = [2, 4]

1.Check: is v₂ = c · v₁ for some scalar c?
2.v₂ = 2 · v₁ = 2[1,2] = [2,4] ✓
3.DEPENDENT! v₂ is just v₁ scaled by 2. These two vectors only span a line, not the full plane.

Now try: v₁ = [1, 0], v₂ = [0, 1]
1.c₁[1,0] + c₂[0,1] = [0,0] means c₁ = 0 and c₂ = 0.
2.INDEPENDENT! Neither is a multiple of the other.

Basis: the minimal spanning set

A basis for a vector space is a set of linearly independent vectors that spans the entire space. In ℝ², you need exactly 2 independent vectors. In ℝⁿ, you need exactly n.

The standard basis for ℝ³ is: e₁ = [1,0,0], e₂ = [0,1,0], e₃ = [0,0,1]. But there are infinitely many valid bases — any 3 independent vectors in ℝ³ work.

IOAI Connection — PCA PCA (Principal Component Analysis) finds a new basis for your data — one where the first basis vector captures the most variance, the second captures the next most, etc. You then keep only the first k basis vectors, reducing dimensions from n to k. This is Day 7 material, but it's built entirely on these concepts.
Section 04

What Is a Matrix?

A matrix is a rectangular grid of numbers. An m×n matrix has m rows and n columns:

A 2×3 matrix (2 rows, 3 columns) A = ⎡ 1   2   3 ⎤
    ⎣ 4   5   6 ⎦

But there's a much deeper way to think about this. 3Blue1Brown's key insight:

The BIG idea (3B1B Video 3) A matrix IS a transformation. Its columns tell you where the basis vectors land after the transformation. When you multiply a matrix by a vector, you're applying that transformation to the vector.

Matrix-vector multiplication

When you compute Av, you're writing v as a linear combination of A's columns:

Matrix-vector product = linear combination of columns ⎡ 2   1 ⎤ ⎡ 3 ⎤  =  3·⎡ 2 ⎤ + 1·⎡ 1 ⎤  =  ⎡ 7 ⎤
⎣ 0   3 ⎦ ⎣ 1 ⎦       ⎣ 0 ⎦     ⎣ 3 ⎦     ⎣ 3 ⎦

Equivalently, using the row view (dot each row with v):

Row-by-row computation:

A = [[2, 1], [0, 3]],   v = [3, 1]

1.Row 1 · v = [2, 1] · [3, 1] = 2×3 + 1×1 = 7
2.Row 2 · v = [0, 3] · [3, 1] = 0×3 + 3×1 = 3
3.Result: [7, 3] ✓

Matrix-matrix multiplication

AB means "first apply B, then apply A." It's function composition for transformations. To compute AB: each column of the result is A times the corresponding column of B.

Compute AB by hand:

A = [[1, 2], [3, 4]],    B = [[5, 6], [7, 8]]

1.Column 1 of AB = A · [5, 7] = [1·5+2·7, 3·5+4·7] = [19, 43]
2.Column 2 of AB = A · [6, 8] = [1·6+2·8, 3·6+4·8] = [22, 50]
3.AB = [[19, 22], [43, 50]]

CRITICAL: AB ≠ BA in general! Matrix multiplication is NOT commutative. 
# Matrix operations in NumPy
A = np.array([[1, 2], [3, 4]])
B = np.array([[5, 6], [7, 8]])
v = np.array([3, 1])

# Matrix-vector product
print(A @ v)        # [5, 13]

# Matrix-matrix product
print(A @ B)        # [[19, 22], [43, 50]]
print(B @ A)        # [[23, 34], [31, 46]]  ← DIFFERENT!

# ⚠️ np.dot and @ do the same thing for 2D arrays
# But for 1D arrays, * does element-wise, not matrix multiply!
print(A * B)        # [[ 5, 12], [21, 32]]  ← ELEMENT-WISE, not matrix multiply!
Deadly NumPy trap A * B does element-wise multiplication (Hadamard product), NOT matrix multiplication. Always use A @ B or np.dot(A, B) for true matrix products. This mistake will cost you hours of debugging in IOAI coding problems.
Section 05

Matrices as Transformations (The Geometric View)

This is the single most important idea in 3Blue1Brown's series. A 2×2 matrix takes every point in 2D space and moves it to a new location. The matrix columns tell you where [1,0] and [0,1] end up.

Identity (no change)

[[1,0],[0,1]] — basis vectors stay put. Every point stays where it is.

Scaling

[[2,0],[0,3]] — stretch x by 2, y by 3. The unit square becomes a 2×3 rectangle.

Rotation 90°

[[0,-1],[1,0]] — î goes to [0,1], ĵ goes to [-1,0]. Everything rotates counter-clockwise.

Shear

[[1,1],[0,1]] — î stays at [1,0], ĵ goes to [1,1]. Tilts the space sideways.

How to read any 2×2 matrix geometrically:

Take A = [[3, 1], [0, 2]]

1.Column 1 = [3, 0] → This is where î = [1,0] lands
2.Column 2 = [1, 2] → This is where ĵ = [0,1] lands
3.The unit square (corners at [0,0], [1,0], [0,1], [1,1]) becomes a parallelogram with corners at [0,0], [3,0], [1,2], [4,2]
4.The area of that parallelogram = |det(A)| = |3·2 - 1·0| = 6 (we'll cover this next!)
import matplotlib.pyplot as plt
import numpy as np

def plot_transformation(A, title="Transformation"):
    """Visualize how matrix A transforms the unit square."""
    # Unit square corners
    square = np.array([[0,0], [1,0], [1,1], [0,1], [0,0]]).T
    
    # Transform it
    transformed = A @ square
    
    fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(10, 5))
    
    # Before
    ax1.fill(square[0], square[1], alpha=0.3, color='blue')
    ax1.quiver(0, 0, 1, 0, angles='xy', scale=1, scale_units='xy', color='red', label='î')
    ax1.quiver(0, 0, 0, 1, angles='xy', scale=1, scale_units='xy', color='green', label='ĵ')
    ax1.set_title("Before")
    ax1.set_xlim(-1, 5); ax1.set_ylim(-1, 5)
    ax1.set_aspect('equal'); ax1.grid(True); ax1.legend()
    
    # After
    ax2.fill(transformed[0], transformed[1], alpha=0.3, color='orange')
    ax2.quiver(0,0, A[0,0],A[1,0], angles='xy', scale=1, scale_units='xy', color='red', label='Aî')
    ax2.quiver(0,0, A[0,1],A[1,1], angles='xy', scale=1, scale_units='xy', color='green', label='Aĵ')
    ax2.set_title(title)
    ax2.set_xlim(-1, 5); ax2.set_ylim(-1, 5)
    ax2.set_aspect('equal'); ax2.grid(True); ax2.legend()
    plt.show()

# Try these!
A = np.array([[3, 1], [0, 2]])
plot_transformation(A, "A = [[3,1],[0,2]]")
Section 06

Determinants — The Area Scaling Factor

The determinant of a matrix tells you how much the transformation scales area (in 2D) or volume (in 3D).

Determinant of a 2×2 matrix det ⎡ a   b ⎤ = ad − bc
    ⎣ c   d ⎦
det = 6

Areas get multiplied by 6. The transformation stretches space.

det = 1

Areas preserved. Rotation is a common example.

det = -2

Areas scale by 2, but orientation flips (like a mirror). The negative means space is "turned inside out."

det = 0

Space gets squished to a lower dimension — a plane collapses to a line, or a line to a point. The matrix is NOT invertible.

Compute these determinants:

1.A = [[3, 1], [0, 2]] → det = 3·2 − 1·0 = 6
2.B = [[1, 2], [3, 6]] → det = 1·6 − 2·3 = 0 → Not invertible! (Row 2 = 3 × Row 1)
3.Rotation = [[0, -1], [1, 0]] → det = 0·0 − (−1)·1 = 1 → Area preserved ✓

3×3 determinant (cofactor expansion)

For a 3×3 matrix, expand along the first row:

3×3 determinant det ⎡ a   b   c ⎤
    ⎢ d   e   f ⎥ = a(ei−fh) − b(di−fg) + c(dh−eg)
    ⎣ g   h   i ⎦
Example: 3×3 determinant by hand

M = [[2, 1, 0], [0, 3, 1], [1, 0, 2]]

1.= 2·det[[3,1],[0,2]] − 1·det[[0,1],[1,2]] + 0·det[[0,3],[1,0]]
2.= 2·(3·2 − 1·0) − 1·(0·2 − 1·1) + 0
3.= 2·(6) − 1·(−1) + 0
4.= 12 + 1 = 13
# Determinants in NumPy
A = np.array([[3, 1], [0, 2]])
print(np.linalg.det(A))  # 6.0

M = np.array([[2,1,0], [0,3,1], [1,0,2]])
print(np.linalg.det(M))  # 13.0

# Singular matrix (det = 0)
S = np.array([[1, 2], [3, 6]])
print(np.linalg.det(S))  # ≈ 0.0 (might show tiny float error)

Matrix Inverse — Undoing a Transformation

The inverse A⁻¹ is the matrix that "undoes" A. If you apply A then A⁻¹, you get back where you started:

Inverse definition A · A⁻¹ = A⁻¹ · A = I     (where I is the identity matrix)

A matrix is invertible if and only if its determinant is non-zero. If det(A) = 0, the transformation squishes space into a lower dimension — you can't unsquish it.

2×2 inverse formula (memorize this)

2×2 inverse ⎡ a   b ⎤⁻¹  =   1   ⎡  d   −b ⎤
⎣ c   d ⎦      ad−bc ⎣ −c    a ⎦

Recipe: swap a and d, negate b and c, divide by determinant.

Find the inverse of A = [[3, 1], [0, 2]]:

1.det(A) = 3·2 − 1·0 = 6
2.Swap diagonal: [[2, _], [_, 3]], Negate off-diagonal: [[2, −1], [0, 3]]
3.A⁻¹ = (1/6) · [[2, −1], [0, 3]] = [[1/3, −1/6], [0, 1/2]]

Verify: A · A⁻¹ = [[3,1],[0,2]] · [[1/3, −1/6],[0, 1/2]]
4.Row 1: [3·(1/3)+1·0, 3·(−1/6)+1·(1/2)] = [1, −1/2+1/2] = [1, 0] ✓
5.Row 2: [0·(1/3)+2·0, 0·(−1/6)+2·(1/2)] = [0, 1] = [0, 1] ✓
6.Result is identity matrix I = [[1,0],[0,1]]. The inverse is correct!
A = np.array([[3, 1], [0, 2]])

# NumPy inverse
A_inv = np.linalg.inv(A)
print(A_inv)
# [[ 0.333  -0.167]
#  [ 0.      0.5  ]]

# Verify: should be identity
print(A @ A_inv)
# [[1. 0.]
#  [0. 1.]]

# YOUR CODING TASK: Implement 2×2 inverse from scratch
def my_inverse_2x2(A):
    a, b = A[0, 0], A[0, 1]
    c, d = A[1, 0], A[1, 1]
    det = a * d - b * c
    if abs(det) < 1e-10:
        raise ValueError("Matrix is singular (det=0), cannot invert")
    return (1 / det) * np.array([[d, -b], [-c, a]])

# Test it
print(my_inverse_2x2(A))
print(np.allclose(A @ my_inverse_2x2(A), np.eye(2)))  # True
Section 08

Column Space & Null Space

Column space (image): where can A send things?

The column space of A (also called the "image" or "range") is the set of all possible outputs when you multiply A by any vector. It's the span of A's columns.

Example:
A = [[1, 2], [2, 4]]

1.Column 1 = [1, 2], Column 2 = [2, 4] = 2 · [1, 2]
2.Column 2 is just a scaled version of column 1!
3.Column space = span{[1, 2]} = a line through the origin in direction [1, 2]
4.No matter what vector you multiply A by, the output always lands on this line.

Null space (kernel): what gets sent to zero?

The null space is the set of all vectors v such that Av = 0. These vectors get "crushed to nothing" by the transformation.

Find the null space of A = [[1, 2], [2, 4]]:

1.We need Av = 0, i.e., [[1,2],[2,4]] · [x,y] = [0,0]
2.Row 1: x + 2y = 0 → x = −2y
3.Row 2: 2x + 4y = 0 → same equation (2 times row 1)
4.Solution: v = y · [−2, 1] for any y
5.Null space = span{[−2, 1]} = a line in direction [−2, 1]
The rank-nullity theorem dim(column space) + dim(null space) = number of columns
For our 2×2 example: rank 1 + nullity 1 = 2 columns. ✓
This is guaranteed to appear on IOAI theory exams.
Section 09

Rank — How Many Dimensions Does the Output Have?

The rank of a matrix = the dimension of its column space = the number of linearly independent columns. It tells you the "true dimensionality" of the transformation's output.

Full rank (2×2)

Rank = 2. Both columns independent. Maps 2D to 2D. det ≠ 0. Invertible.

Rank 1 (2×2)

One column is a multiple of the other. Maps 2D → a line. det = 0. Not invertible.

Rank 0

The zero matrix. Everything maps to the origin.

3×3, rank 2

Maps 3D space to a plane. One column is a combination of the other two.

Find rank of M = [[1,2,3],[2,4,6],[1,1,1]] without a calculator:

1.Row 2 = 2 × Row 1 → Row 2 is redundant
2.After eliminating Row 2, we have rows [1,2,3] and [1,1,1]
3.These two rows are NOT multiples of each other (independent)
4.Rank = 2
5.Geometric meaning: 3 columns live in a 2D plane inside 3D space, not all of 3D.
# Rank in NumPy
M = np.array([[1,2,3],[2,4,6],[1,1,1]])
print(np.linalg.matrix_rank(M))  # 2

# Full rank matrix
F = np.array([[1,0,0],[0,1,0],[0,0,1]])
print(np.linalg.matrix_rank(F))  # 3

Systems of Linear Equations & Gaussian Elimination

A system of linear equations is just a matrix equation Ax = b. The question: "given transformation A and desired output b, what input x gives us that output?"

System → Matrix form x + 2y − z = 3           ⎡ 1   2  −1 ⎤ ⎡ x ⎤    ⎡ 3 ⎤
2x + y + z = 5     →    ⎢ 2   1   1 ⎥ ⎢ y ⎥ = ⎢ 5 ⎥
x − y + 2z = 1           ⎣ 1  −1   2 ⎦ ⎣ z ⎦    ⎣ 1 ⎦

Gaussian elimination: the systematic method

The idea: use row operations to turn the matrix into an upper triangular form, then back-substitute. The allowed operations are: (1) swap two rows, (2) multiply a row by a non-zero scalar, (3) add a multiple of one row to another.

Solve by Gaussian elimination:
x + 2y − z = 3   (R1)
2x + y + z = 5   (R2)
x − y + 2z = 1   (R3)

Step 1: Eliminate x from R2 and R3
1.R2 ← R2 − 2·R1:  (2−2)x + (1−4)y + (1+2)z = 5−6 → −3y + 3z = −1
2.R3 ← R3 − 1·R1:  (1−1)x + (−1−2)y + (2+1)z = 1−3 → −3y + 3z = −2

Step 2: Eliminate y from R3
3.R3 ← R3 − R2:  0y + 0z = −2−(−1) → 0 = −1

4.CONTRADICTION! 0 = −1 is impossible. This system has no solution.
5.Geometrically: the three planes don't all meet at a single point.

Let me also show a system that does have a solution:

Solve: x + 2y = 5,   3x + y = 10

Augmented matrix: [[1, 2 | 5], [3, 1 | 10]]

1.R2 ← R2 − 3·R1:  [0, 1−6, 10−15] = [0, −5, −5]
2.From R2: −5y = −5 → y = 1
3.Back-substitute into R1: x + 2(1) = 5 → x = 3
4.Solution: (x, y) = (3, 1)
5.Verify: 1(3) + 2(1) = 5 ✓,   3(3) + 1(1) = 10 ✓
# Solving systems in NumPy
A = np.array([[1, 2], [3, 1]])
b = np.array([5, 10])
x = np.linalg.solve(A, b)
print(x)                      # [3. 1.]
print(np.allclose(A @ x, b))  # True

# YOUR TASK: Implement Gaussian elimination from scratch
def gauss_solve(A, b):
    """Solve Ax = b using Gaussian elimination with back-substitution."""
    n = len(b)
    # Build augmented matrix [A | b]
    aug = np.hstack([A.astype(float), b.reshape(-1, 1).astype(float)])
    
    # Forward elimination
    for col in range(n):
        # Find pivot (row with largest absolute value in this column)
        max_row = col + np.argmax(np.abs(aug[col:, col]))
        aug[[col, max_row]] = aug[[max_row, col]]  # swap rows
        
        # Eliminate below
        for row in range(col + 1, n):
            factor = aug[row, col] / aug[col, col]
            aug[row] -= factor * aug[col]
    
    # Back-substitution
    x = np.zeros(n)
    for i in range(n - 1, -1, -1):
        x[i] = (aug[i, -1] - aug[i, :-1] @ x) / aug[i, i]
    return x

# Test
x = gauss_solve(A, b)
print(x)  # [3. 1.]

Eigenvalues & Eigenvectors — The Most Important Concept

This is the crown jewel of Day 1 and one of the most important concepts in all of ML. Here's the idea in plain English:

The core idea Most vectors change direction when you apply a matrix transformation. But some special vectors only get stretched or compressed — they stay on the same line. These are eigenvectors. The factor by which they get scaled is the eigenvalue.
Eigenvalue equation Av = λv

Read this as: "when I apply transformation A to vector v, the result is just v scaled by λ." Here, v is the eigenvector and λ (lambda) is the eigenvalue.

How to find eigenvalues: the characteristic equation

Starting from Av = λv, rearrange:

Derivation Av = λv
Av − λv = 0
(A − λI)v = 0

For a non-zero v to exist, (A − λI) must be singular:

det(A − λI) = 0 ← This is the characteristic equation!
Find eigenvalues and eigenvectors of A = [[3, 1], [0, 2]]:

Step 1: Characteristic equation
1.A − λI = [[3−λ, 1], [0, 2−λ]]
2.det(A − λI) = (3−λ)(2−λ) − (1)(0) = (3−λ)(2−λ)
3.Set = 0: (3−λ)(2−λ) = 0
4.λ₁ = 3, λ₂ = 2

Step 2: Find eigenvector for λ₁ = 3
5.(A − 3I)v = 0 → [[0, 1], [0, −1]] · [x, y] = [0, 0]
6.Row 1: 0·x + 1·y = 0 → y = 0
7.x is free. So v₁ = t·[1, 0] for any t ≠ 0. Pick v₁ = [1, 0]

Step 3: Find eigenvector for λ₂ = 2
8.(A − 2I)v = 0 → [[1, 1], [0, 0]] · [x, y] = [0, 0]
9.Row 1: x + y = 0 → x = −y
10.Pick y = 1, then x = −1. v₂ = [−1, 1]

Step 4: Verify
11.Av₁ = [[3,1],[0,2]] · [1,0] = [3, 0] = 3 · [1, 0] = λ₁ · v₁ ✓
12.Av₂ = [[3,1],[0,2]] · [−1,1] = [−3+1, 0+2] = [−2, 2] = 2 · [−1, 1] = λ₂ · v₂ ✓

Shortcut for 2×2 eigenvalues

For a 2×2 matrix [[a,b],[c,d]], the characteristic equation is always a quadratic:

2×2 eigenvalue formula λ² − (a+d)λ + (ad−bc) = 0

λ² − trace(A)·λ + det(A) = 0

where trace(A) = a + d (sum of diagonal)

Then use the quadratic formula. For A = [[3,1],[0,2]]: λ² − 5λ + 6 = 0 → (λ−3)(λ−2) = 0 → λ = 3, 2. ✓

A = np.array([[3, 1], [0, 2]])

# Verify: A @ v = lambda * v
for i in range(2):
    v = eigenvectors[:, i]   # i-th eigenvector is i-th COLUMN
    lam = eigenvalues[i]
    print(f"A @ v{i} = {A @ v}")
    print(f"λ{i} * v{i} = {lam * v}")
    print(f"Match: {np.allclose(A @ v, lam * v)}")
    print()
Why IOAI cares about eigenvalues Eigenvalues are EVERYWHERE in ML:
  • PCA: The principal components ARE the eigenvectors of the covariance matrix. The eigenvalues tell you how much variance each component captures.
  • Google PageRank: The ranking of web pages is the dominant eigenvector of the link matrix.
  • Stability of training: If the eigenvalues of the Hessian matrix of your loss are too large, gradient descent will diverge.
  • Kernel methods: Proving a kernel is valid requires showing all eigenvalues of the Gram matrix are ≥ 0 (positive semi-definite).
IOAI 2024 had a theory question asking to prove rank(AᵀA) = rank(A), which requires understanding null spaces and eigenvalues.

What eigenvalues tell you geometrically

λ > 1

The eigenvector gets stretched. The transformation expands along this direction.

0 < λ < 1

The eigenvector gets compressed. The transformation shrinks along this direction.

λ = 0

The eigenvector gets sent to zero. This direction is in the null space. Matrix is singular.

λ < 0

The eigenvector gets flipped and scaled. Direction reverses.

3.A⁻¹(AB) = A⁻¹(AC)

IOAI-Style Problem: Prove rank(AᵀA) = rank(A)

Click to see proof (this is the kind of problem IOAI asks!)
Strategy: Show Ax = 0 and AᵀAx = 0 have the same solutions. Then by rank-nullity, they have the same rank.

Direction 1: If Ax = 0, then AᵀAx = 0
1.If Ax = 0, then Aᵀ(Ax) = Aᵀ · 0 = 0
2.So AᵀAx = 0 ✓

Direction 2: If AᵀAx = 0, then Ax = 0
3.Assume AᵀAx = 0
4.Left-multiply by xᵀ: xᵀAᵀAx = 0
5.Notice: xᵀAᵀ = (Ax)ᵀ, so (Ax)ᵀ(Ax) = 0
6.This equals ‖Ax‖² = 0 (the squared length of Ax)
7.A squared length can only be 0 if the vector itself is 0, so Ax = 0 ✓

Conclusion:
8.null(A) = null(AᵀA) → they have the same nullity
9.Both A and AᵀA have the same number of columns (n)
10.By rank-nullity: rank = (# columns) − nullity
11.Same # columns, same nullity → same rank ∎
Why this matters for ML This proof is essential because in least-squares regression, we solve AᵀAx = Aᵀb (the "normal equation"). This result guarantees that the normal equation loses no information compared to the original system Ax = b.

import numpy as np
import matplotlib.pyplot as plt

def full_visualizer(A):
    """Complete Day 1 coding task: visualize matrix transformation + eigenvectors."""
    # Unit square
    square = np.array([[0,1,1,0,0], [0,0,1,1,0]])
    transformed = A @ square
    
    # Eigenstuff
    vals, vecs = np.linalg.eig(A)
    
    fig, axes = plt.subplots(1, 2, figsize=(12, 6))
    
    for ax in axes:
        ax.set_xlim(-4, 6); ax.set_ylim(-4, 6)
        ax.set_aspect('equal'); ax.grid(True, alpha=0.3)
        ax.axhline(y=0, color='gray', lw=0.5)
        ax.axvline(x=0, color='gray', lw=0.5)
    
    # Before
    axes[0].fill(square[0], square[1], alpha=0.2, color='steelblue')
    axes[0].plot(square[0], square[1], 'steelblue', lw=2)
    axes[0].set_title('Original Unit Square', fontsize=14)
    
    # After + eigenvectors
    axes[1].fill(transformed[0], transformed[1], alpha=0.2, color='coral')
    axes[1].plot(transformed[0], transformed[1], 'coral', lw=2)
    
    colors = ['#2ecc71', '#9b59b6']
    for i in range(len(vals)):
        if np.isreal(vals[i]):  # Only plot real eigenvectors
            v = vecs[:, i].real
            lam = vals[i].real
            # Draw eigenvector direction
            axes[1].annotate('', xy=v*lam, xytext=(0,0),
                arrowprops=dict(arrowstyle='->', color=colors[i], lw=2.5))
            axes[1].annotate('', xy=v*3, xytext=v*-3,
                arrowprops=dict(arrowstyle='-', color=colors[i], lw=1, ls='--'))
            axes[1].text(v[0]*lam+0.2, v[1]*lam+0.2,
                f'λ={lam:.1f}', color=colors[i], fontsize=12, fontweight='bold')
    
    axes[1].set_title(f'After A · det={np.linalg.det(A):.1f}', fontsize=14)
    plt.tight_layout()
    plt.show()

# Run it!
A = np.array([[3, 1], [0, 2]])
full_visualizer(A)

# independent columns. Rank + Nullity = # columns. Full rank = invertible.

Row operations to solve Ax=b. Forward eliminate, then back-substitute.

Av=λv. Find λ: det(A−λI)=0. Find v: solve (A−λI)v=0. Foundation of PCA.
